## Salesforce Custom Objects

This project uses multiple custom objects to store employee data, feedback insights, referral information, and event management records.  
These objects act as the core data layer for automation flows, AI prompts, and Agentforce agents.

---

## 1. Employee Object

The **Employee** object stores all essential employee profile information and engagement indicators.  
It is the primary data source used by automation flows and AI agents.

### Key Fields

| Field | Type | Description |
|------|------|-------------|
| Employee ID | Auto Number | Unique identifier for each employee |
| Name | Text | Employee name |
| Email | Email (Unique) | Employee email address |
| Department | Picklist | Department the employee belongs to |
| Role | Text | Job role of the employee |
| Location | Text | Employee working location |
| Joining Date | Date | Date when the employee joined the company |
| Birthday | Date | Employee birth date used for celebrations |
| Interest | Text | Employee interests |
| Hobbies | Text | Employee hobbies |
| Skill | Text | Employee skill set |
| Engagement Score | Number | Calculated engagement level |
| Burnout Risk | Picklist | Burnout risk category (Low / Medium / High) |
| Burnout Analysis | Long Text | AI generated burnout insights |
| AI Summary | Long Text | Overall AI-generated summary of employee status |
| Birthday Email Message | Long Text | Personalized birthday message generated by the system |

### Usage

This object is used by:

- Burnout Detection Flow  
- Birthday Assistant Agent  
- Engagement analysis automation  
- HR insights and reporting

---

## 2. Referral Object

The **Referral** object stores information about candidates referred by employees.

### Key Fields

| Field | Type | Description |
|------|------|-------------|
| Referral Name | Text | Name of the referred candidate |
| Age | Number | Candidate age |
| Experience | Number | Candidate work experience |
| Role | Text | Role the candidate is referred for |

### Usage

This object supports the **Referral Submission Flow**, allowing employees to recommend potential candidates for open positions.

---

## 3. Feedback Object

The **Feedback** object stores employee feedback data and sentiment analysis results.

### Key Fields

| Field | Type | Description |
|------|------|-------------|
| Feedback ID | Auto Number | Unique identifier for feedback |
| Employee | Lookup(Employee) | Employee who submitted feedback |
| Feedback Text | Long Text | Feedback content submitted by employee |
| Category | Picklist | Feedback category |
| Date | Date | Feedback submission date |
| Sentiment | Text | Sentiment classification (Positive / Neutral / Negative) |
| Sentiment Score | Number | AI-generated sentiment score |
| Engagement Score | Number | Updated engagement score based on feedback |

### Usage

This object is used in:

- Feedback Sentiment Analysis Flow  
- Engagement Score calculation  
- Burnout risk analysis

---

## 4. Event Object

The **Event** object manages company events and employee engagement activities.

### Key Fields

| Field | Type | Description |
|------|------|-------------|
| Event Name | Text | Name of the event |
| Event Type | Picklist | Type of event (e.g., networking, celebration) |
| Theme | Text | Event theme |
| Description | Long Text | Event details |
| Date | Date | Event date |
| Location | Text | Event location |
| Capacity | Number | Maximum number of participants |

### Usage

This object supports event-related automations such as:

- Event Management Agent  
- Networking Lunch Agent  
- Employee engagement activities

---

The objects interact with each other to support HR automation workflows.

Example relationship:

Employee

↓

↓  
Engagement Score Update  
↓  
Burnout Detection Agent  

Employee  
↓  
Event Participation  

Employee  
↓  
Referral Submission  

---

## Role in System Architecture

These objects form the **data layer** of the system and are used by:

- Salesforce automation flows
- AI prompts for analysis
- Agentforce agents for decision making
- HR dashboards and notifications

They enable the system to collect structured employee data, perform AI analysis, and trigger automated HR actions.
